# Contribuer a OrchivisteKit

Merci de contribuer.

## Principes

- Portee: priorite a la coherence documentaire, structurelle et de publication.
- Pas de refactor metier majeur sans decision explicite.
- Respect du role du depot dans l'ecosysteme Orchiviste/Muni.

## Workflow

- Branche de travail: `feature/*`, `fix/*`, `docs/*`, `chore/*`.
- Commits atomiques et messages explicites.
- Ouvrir une Pull Request avec contexte, changements, risques et validations.

## Versionnage

- Ce depot suit Semantic Versioning.
- Documenter les evolutions dans `CHANGELOG.md`.

## Licence

En contribuant, vous acceptez que votre contribution soit publiee sous GNU GPL v3.0.

## Verification minimale

- Verifier la coherence des README/docs lies au changement.
- Verifier qu'aucun ancien nom de projet n'est introduit dans la documentation publique.
